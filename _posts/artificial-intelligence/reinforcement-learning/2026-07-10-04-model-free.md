第三章的 DP 方法需要完整的环境模型 $p(s', r \mid s, a)$。现实中这几乎永远不可行——你不知道物理世界的精确动力学，也不知道金融市场的转移概率。本章介绍**无模型 (Model-Free) 方法**，只通过与环境的**交互采样**来学习，不需要知道 $p(s', r \mid s, a)$。

---

## 4.1 为什么需要无模型？

### 模型在现实中遥不可及

计算或估计 $p(s', r \mid s, a)$ 通常代价高昂甚至不可能：
- **机器人控制**：无法写出摩擦、惯性、传感器噪声的精确联合分布
- **自动驾驶**：其他车辆和行人的行为无法用固定概率表描述
- **围棋 / 游戏**：对手的策略未知且动态变化

### 无模型设定：环境模拟器

在无模型设定下，我们不需要模型的解析形式，只需要一个**黑箱模拟器**：

$$(s', r) = \text{EnvironmentSimulator}(s, a) \sim p(\cdot, \cdot \mid s, a)$$

> 模拟器比完整模型**弱得多**——它只能被"查询"（输入 $s, a$，输出 $s', r$），而不能被"分析"（不能求导、不能反向推演）。

## 4.2 Monte Carlo 策略评估

### 回到价值的原始定义

在模型已知时，我们通过求解 Bellman 方程来评估策略：

$$V^{\pi} = (I - \gamma P(\pi))^{-1} b(\pi)$$

这需要 $P$ 和 $b$——但在无模型设定下，我们不知道它们。

**替代方案**：回到价值函数的**定义**：

$$V^{\pi}(s) = \mathbb{E}_{\pi}\left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_0 = s \right]$$

> 我们不知道转移概率，但我们知道**如何采样**。Monte Carlo 方法的核心：用大量采样的平均值来近似期望值。

### MC 策略评估算法

对每个需要评估的状态 $s$，遵循策略 $\pi$ 从 $s$ 出发运行 $N$ 条完整轨迹（episode）。对第 $n$ 条轨迹：

$$S_0^{(n)} = s,\quad A_0^{(n)} \sim \pi(\cdot \mid S_0^{(n)}),\quad R_1^{(n)},\quad S_1^{(n)},\quad A_1^{(n)} \sim \pi(\cdot \mid S_1^{(n)}),\quad R_2^{(n)},\ \ldots$$

计算该轨迹从 $s$ 开始的折扣回报：

$$G^{(n)} = \sum_{k=0}^{\infty} \gamma^k R_{k+1}^{(n)}$$

取 $N$ 条轨迹回报的平均作为价值估计：

$$V^{\pi}(s) \approx \frac{1}{N} \sum_{n=1}^{N} G^{(n)}$$

> **关键特征**：必须等到 episode 结束才能计算 $G^{(n)}$，因为需要完整的奖励序列。这是 MC 方法最大的限制——对于没有终点的持续任务 (continuing tasks)，MC 无法直接使用。

### MC 策略改进

同样地，估计 $Q^{\pi}(s, a)$ 然后贪心改进策略：

$$Q^{\pi}(s, a) \approx \frac{1}{N} \sum_{n=1}^{N} \left[ \sum_{k=0}^{\infty} \gamma^k R_{k+1}^{(n)} \right], \quad S_0^{(n)} = s, A_0^{(n)} = a$$

$$\pi'(s) = \arg\max_a Q^{\pi}(s, a)$$

### 无模型策略迭代算法

**输入**：环境模拟器、采样数 $N$、停止准则；**初始化**：任意策略 $\pi$，$Q(s, a) \leftarrow 0\ (\forall s, a)$。

重复以下两步直至策略不再更新：

**(1) 策略评估** —— 对每个 $(s, a)$ 用 MC 估计 $Q^{\pi}(s, a)$：

$$\forall s \in S,\ \forall a \in A(s): \quad Q(s, a) \leftarrow \frac{1}{N} \sum_{n=1}^{N} \left[ \sum_{k=0}^{\infty} \gamma^k R_{k+1}^{(n)} \right]$$

其中第 $n$ 条 episode 从 $(S_0 = s, A_0 = a)$ 出发，遵循 $\pi$ 生成。

**(2) 策略改进** —— 贪心：

$$\forall s \in S: \quad \pi(s) \leftarrow \arg\max_{a} Q(s, a)$$

**输出**：$\pi \approx \pi^*,\ Q \approx Q^*$

> 此算法的复杂度极高——对**每个** (s, a) 对都要运行 N 条完整 episode。这在实际中几乎不可行，但它是连接 DP 策略迭代和无模型学习的理论桥梁。

---

## 4.3 原始 MC 方法的致命缺陷

尽管概念简洁，原始 MC 方法在实践中面临一系列严重问题：

| 问题 | 原因 | 后果 |
|------|------|------|
| **延迟更新** | 必须等 episode 结束才能计算回报 | 长 episode 中学习极其缓慢 |
| **高方差** | 回报 $G_t$ 涉及 $\sum \gamma^k r_{t+k}$，方差随 episode 长度累积 | 学习不稳定，需要大量样本 |
| **样本低效** | 每次访问状态只积累一条回报 | 需要海量 episode 才能收敛 |
| **信用分配差** | 同一条 episode 中的所有动作共享同一个回报 | 无法区分哪些动作是关键，哪些是噪音 |

> 这些缺陷催生了时差学习 (TD Learning)——RL 中最重要的算法创新。

---

## 4.4 时差学习 (TD Learning)：核心思想

### 融合两种方法的优点

| | Monte Carlo (MC) | 动态规划 (DP) | 时差学习 (TD) |
|---|---|---|---|
| 需要模型？ | 否 | 是 | **否** |
| 何时更新？ | Episode 结束 | 每步 | **每步** |
| 用 Bootstrapping？ | 否（真实回报） | 是（模型推断） | **是（采样推断）** |

**TD 的创新**：不等到 episode 结束，也不依赖模型——**用采样的下一状态和已学到的价值来构造学习目标**。这叫 **Bootstrapping（自举）**。

### 两种学习目标

- **MC 目标**（完整真实回报）：$G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots$
- **TD 目标**（一步自举）：$G_t^{(1)} = r_t + \gamma V(s_{t+1})$

TD 用对 $V(s_{t+1})$ 的当前估计来**替换** $r_{t+1} + \gamma r_{t+2} + \cdots$ 这串未知的未来。

### 为什么叫"时差"？

从 Bellman 期望方程的**不动点迭代**视角出发：

$$V(s) \leftarrow (1 - \alpha) V(s) + \alpha \left( \mathbb{E}_{a \sim \pi, s' \sim p} [r + \gamma V(s')] \right)$$

TD 方法把期望替换为**单次采样**（类似 SGD 替换 GD）：

$$V(s) \leftarrow V(s) + \alpha \underbrace{\left[ r + \gamma V(s') - V(s) \right]}_{\text{TD 误差 } \delta_t}$$

括号中的差 $\delta_t = r + \gamma V(s') - V(s)$ 就是 **TD 误差 (Temporal Difference error)**。

---

## 4.5 TD 误差的深层含义

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

**解读**：
- $V(s_t)$：你**之前**认为状态 $s_t$ 的价值是多少
- $r_t + \gamma V(s_{t+1})$：你**现在**（观测到一步真实结果后）认为状态 $s_t$ 的价值应该是多少
- $\delta_t$：两者之差——你的预测有多"出乎意料"

> **如果 $\delta_t > 0$**：实际结果比预期好 → 上调 $V(s_t)$
>
> **如果 $\delta_t < 0$**：实际结果比预期差 → 下调 $V(s_t)$
>
> **如果 $\delta_t = 0$**：预测完全准确 → 无需调整

### 时差的直观命名

$$
\underbrace{r_t + \gamma V(s_{t+1})}_{\text{新预测}} \quad \text{v.s.} \quad \underbrace{V(s_t)}_{\text{旧预测}}
$$

"Temporal Difference" 直译是"时间差分"——比较的是**不同时间点**对同一状态的两次估值。

---

## 4.6 TD(0)：一步 TD 算法

### 更新规则

$$V(s_t) \leftarrow V(s_t) + \alpha [r_t + \gamma V(s_{t+1}) - V(s_t)]$$

### TD(0) 策略评估算法

**输入**：环境模拟器、初始状态采样器、策略 $\pi$、学习率 $\alpha$、episode 长度 $T$

**初始化**：$V(s) \leftarrow 0,\ \forall s \in S$

对每个 episode：

$$\begin{aligned}
\text{(1)}\quad & s \leftarrow \text{InitialStateSampler}() \\[4pt]
\text{(2)}\quad & \text{for } t = 0, 1, \ldots, T-1: \\[4pt]
&\quad a \sim \pi(\cdot \mid s) \\[4pt]
&\quad s', r \leftarrow \text{EnvironmentSimulator}(s, a) \\[4pt]
&\quad V(s) \leftarrow (1 - \alpha) V(s) + \alpha \big[ r + \gamma V(s') \big] \\[4pt]
&\quad s \leftarrow s'
\end{aligned}$$

等价的增量形式（更常用）：

$$V(s) \leftarrow V(s) + \alpha \big[ r + \gamma V(s') - V(s) \big]$$

> **与 MC 的关键区别**：TD(0) 每步更新，不需要等到 episode 结束。即使 episode 很长或无限长，也能持续学习。

### TD(0) 的收敛条件

在表格化表示下，若 TD(0) 迭代收敛，对其两边取环境期望可得 $V \to V^{\pi}$。更严格地，确保收敛的条件为：

1. **充分访问**：每个状态被访问无穷多次
2. **Robbins-Monro 学习率条件**：$\sum_{t=0}^{\infty} \alpha_t = \infty$（学习率足够大以克服初始误差）且 $\sum_{t=0}^{\infty} \alpha_t^2 < \infty$（学习率最终衰减以确保收敛）

实践中常用 $\alpha_t = \frac{1}{t}$ 或 $\alpha_t = \frac{c}{c + t}$ 来满足上述条件。在固定学习率 $\alpha$ 下，TD(0) 收敛到 $V^{\pi}$ 的一个邻域而非精确值。

具体来说，设 $\bar{V}$ 为 TD 不动点（即 $\bar{V} = \mathbb{E}[r + \gamma \bar{V}(s') \mid s, \pi]$），$V_k$ 为第 $k$ 步迭代后的估值。在 $\|\cdot\|_{\infty}$ 下，若 $\alpha$ 充分小，迭代序列 $\{V_k\}$ 在均方意义下收敛到 $\bar{V}$ 附近。这直接源于 Bellman 算子的 $\gamma$-压缩性——以 $\gamma$ 的速率收缩误差。

### MC vs TD 的偏差-方差权衡

| | MC | TD(0) |
|---|---|---|
| **是否无偏** | ✅ 是（使用真实样本回报） | ❌ 否（$V(s')$ 本身有误差） |
| **方差** | 高（累积多步随机奖励） | **低**（只依赖一步随机奖励） |
| **收敛速度（实践）** | 慢（方差太大） | **快**（低方差） |
| **对初始值敏感** | 不敏感 | 敏感（bootstrapping 导致初期有偏） |

> **MC 是无偏但高方差，TD 是有偏但低方差。** 实践中 TD 的低方差通常更有价值——它使学习更稳定、更快速。这就是为什么 TD 成为现代 RL 的基石。

## 4.7 Sarsa：On-Policy TD Control

上一节的 TD(0) 评估的是**状态价值** $V(s)$。要改进策略，我们需要**动作价值** $Q(s, a)$。

### 从 V 到 Q

$Q^{\pi}$ 的 Bellman 期望方程：

$$Q^{\pi}(s, a) = \mathbb{E}_{s', r \sim p} \mathbb{E}_{a' \sim \pi} \left[ r + \gamma Q^{\pi}(s', a') \right]$$

TD 版本——用单次采样替代期望：

$$\boxed{Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma Q(s', a') - Q(s, a) \right]}$$

### 名字的由来——(s, a, r, s', a')

```
 Sarsa = State → Action → Reward → State → Action
          s    →   a    →   r    →   s'   →   a'
```

$a'$ 是从当前策略中**实际采样**出来的——因此 Sarsa 是 **On-Policy（同策略）** 方法。

### Sarsa with ε-greedy 算法

**参数**：学习率 $\alpha \in (0, 1)$，折扣因子 $\gamma \in [0, 1)$，探索率 $\varepsilon$

**初始化**：$Q(s, a) \leftarrow 0,\ \forall s \in S, a \in A$

对每个 episode：

$$\begin{aligned}
\text{(1)}\quad & \text{初始化状态 } s \\[4pt]
\text{(2)}\quad & a \leftarrow \begin{cases}
\arg\max_{a'} Q(s, a'), & \text{以概率 } 1 - \varepsilon \\
\text{随机动作}, & \text{以概率 } \varepsilon
\end{cases} \\[4pt]
\text{(3)}\quad & \text{重复直到 } s \text{ 终止}: \\[4pt]
&\quad \text{执行 } a \text{，观测 } r, s' \\[4pt]
&\quad a' \leftarrow \begin{cases}
\arg\max_{a''} Q(s', a''), & \text{以概率 } 1 - \varepsilon \\
\text{随机动作}, & \text{以概率 } \varepsilon
\end{cases} \\[4pt]
&\quad Q(s, a) \leftarrow Q(s, a) + \alpha \big[ r + \gamma Q(s', a') - Q(s, a) \big] \\[4pt]
&\quad s \leftarrow s',\ a \leftarrow a'
\end{aligned}$$

### On-Policy 的含义

Sarsa 的"同策略"特性意味着：
- **行为策略**（用于探索的 ε-greedy）和**目标策略**（正在学习改进的策略）是**同一个**
- 更新目标 $r + \gamma Q(s', a')$ 中的 $a'$ 是 $\varepsilon$-greedy 下采样的，包含了探索噪声
- 因此 Sarsa 会学习一个**考虑了探索成本**的策略——在危险环境中更保守、更安全

## 4.8 Q-Learning：Off-Policy TD Control

### 核心思想

Q-Learning 直接瞄准 $Q^{*}$（最优动作价值），不管当前探索时用了什么策略。

**更新规则**：

$$\boxed{Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \right]}$$

### Sarsa vs Q-Learning：形式上的微小差异，思想上的巨大鸿沟

| | Sarsa (On-Policy) | Q-Learning (Off-Policy) |
|---|---|---|
| 更新目标中的下一动作 | $Q(s', a')$——**实际执行的** | $\max_{a'} Q(s', a')$——**最优的** |
| 学习的是什么策略 | 当前探索策略的 $Q^{\pi}$ | 最优策略的 $Q^{*}$ |
| 探索的影响 | 会被"学到"（保守） | 不被学到（乐观） |
| 稳定性 | 较高 | 较低（可能振荡） |

### Q-Learning 算法

**参数**：学习率 $\alpha \in (0, 1)$，折扣因子 $\gamma \in [0, 1)$，探索率 $\varepsilon$

**初始化**：$Q(s, a) \leftarrow 0,\ \forall s \in S, a \in A$

对每个 episode：

$$\begin{aligned}
\text{(1)}\quad & \text{初始化状态 } s \\[4pt]
\text{(2)}\quad & \text{重复直到 } s \text{ 终止}: \\[4pt]
&\quad a \leftarrow \begin{cases}
\arg\max_{a'} Q(s, a'), & \text{以概率 } 1 - \varepsilon \quad (\text{利用}) \\
\text{随机动作}, & \text{以概率 } \varepsilon \quad (\text{探索})
\end{cases} \\[4pt]
&\quad \text{执行 } a \text{，观测 } r, s' \\[4pt]
&\quad Q(s, a) \leftarrow Q(s, a) + \alpha \big[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \big] \\[4pt]
&\quad s \leftarrow s'
\end{aligned}$$

> **关键洞察**：Q-Learning 的行为策略（ε-greedy）用于探索，但学习目标用的是 $\max$——即最优策略。这就是"Off-Policy"（离策略）的含义——**学习的目标策略和行为策略是分离的**。

## 4.9 Sarsa vs Q-Learning 深度对比

### 悬崖行走示例

考虑一个经典的悬崖行走 (Cliffwalk) 环境：智能体在网格中行走，网格边缘是悬崖，掉落会获得巨大的负奖励。

| 算法 | 学到的行为 | 原因 |
|------|-----------|------|
| **Q-Learning** | 贴着悬崖走最短路（最优但危险） | 学习时假设后续会走最优路径，忽略了探索时可能失足 |
| **Sarsa** | 远离悬崖绕远路（安全但保守） | 学习时考虑到了 ε-greedy 探索可能导致的"失足" |

> **Q-Learning 学"最优路径"，Sarsa 学"安全的路径"。** 在实际部署中，如果探索无法完全关闭（如在线学习），Sarsa 的保守性是一个优点。

### 选择指南

| 场景 | 推荐算法 |
|------|----------|
| 离线批量学习，有大量历史数据 | Q-Learning（Off-Policy 可从任意数据学习） |
| 在线学习，安全至关重要 | Sarsa（更稳定、更保守） |
| 需要重用旧策略的经验 | Q-Learning（Off-Policy） |
| 环境有高风险状态 | Sarsa（On-Policy 更安全） |

## 4.10 MC / TD / DP 全面对比

| 维度 | MC | TD | DP |
|------|-----|----|----|
| **需要模型** | 否 | 否 | 是 |
| **更新时间** | Episode 结束后 | 每步 | 每步（扫描所有状态） |
| **Bootstrapping** | 否 | 是 | 是 |
| **方差** | 高 | 中 | 低（有模型时精确） |
| **偏差** | 无偏 | 有偏 | 无偏（有模型时精确） |
| **数据效率** | 低 | 高 | N/A（用模型不用数据） |
| **处理持续任务** | 困难 | 容易 | 容易 |
| **实用性** | 差（仅作理论参考） | **好**（实用方法的基础） | 受限于模型可用性 |

## 小结

- **无模型方法** 通过环境模拟器的采样来学习，绕过了对转移概率 $p(s', r \mid s, a)$ 的依赖。
- **Monte Carlo** 用完整回报来估计价值——无偏但方差极大，必须等 episode 结束。
- **时差学习 (TD)** 用 bootstrapping（$r + \gamma V(s')$）替代未知的未来——有偏但低方差，且每步更新。TD 是现代 RL 的引擎。
- **Sarsa** 是 On-Policy TD——用实际采样的下一动作构造目标，学习保守策略。
- **Q-Learning** 是 Off-Policy TD——用 $\max$ 构造目标，直接学习最优策略。
- 当状态/动作空间变得很大时，表格化 Q 表示不再可行——下一章将介绍**价值函数逼近**和 Deep Q-Network。
